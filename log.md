## v1.4.1 — 2026-07-01

### ヒントドットの視認性改善
- 実機（iPad）で確認したところ、ヒントドットの半径 `0.2`・最大不透明度 `0.7` が控えめすぎてほぼ気づけなかったため、半径スケールを `0.2` → `0.8` に拡大（`js/fretboard.js` `_drawCircle()` 呼び出し）。候補ポジションが近接している場合は隣接するドット同士が重なる可能性があるが、実機確認の範囲では視認性を優先
- `guide.html` の表現を「うっすら」→「半透明の大きめの丸」に修正

---

## v1.4.0 — 2026-07-01

### バグ修正・ヒント表示機能を追加
- **ハンバーガーメニュー表示バグの修正**: メニューを開いたままLVボタンをタップするとメニューが開いたままゲーム画面に遷移し、画面の一部が隠れる不具合を修正。`startGame()` の先頭で `menuPanel` の `open` クラスを外すよう変更（`js/app.js`）
- **詰まった時のヒント表示機能を追加**（LV.Max以外の全LV・練習モード対象）:
  - フェーズ1（Root待ち）: 出題音再生完了直後から5〜7秒（ランダム）無反応でタイムアウトし、表示範囲内・判定対象弦の中でルートのピッチクラスに一致する全ポジションに半透明ドットをフェードイン表示
  - フェーズ2（度数音待ち）: Rootタップ確定直後から5〜7秒無反応でタイムアウトし、同様にターゲット音の全ポジションを表示
  - ヒントの使用有無はスコア・ランキングに一切影響しない（記録しない）
  - `js/fretboard.js`: `showHints()` / `clearHints()` を追加。`requestAnimationFrame` で約400msかけてフェードイン。`_drawCircle()` に半径スケール・不透明度の引数を追加（既存呼び出しは後方互換）
  - `js/game.js`: `_scheduleHint()` でスケジュール時点のフェーズを記憶し、フェーズが変わっていたら発火時に無視（「もう一度」で度数フェーズ中に音を聞き直した際の誤発火防止）。ルート正解時・度数正解時・新しい問題開始時・ゲーム停止時にヒントとタイマーをクリア
  - ハンバーガーメニューに「ヒント」ON/OFFトグルを追加（デフォルトON、`js/app.js` / `index.html`）
- `guide.html` にヒント機能の説明を追記
- **練習モードの次問移行タイマーを統一**: 練習モードのみ正解後の次問移行が `800ms`（通常LVは `500ms`）となっており、`guide.html` の「0.5秒後に自動で次の問題へ進みます」という記述と食い違っていた。意図的な設計差ではなかったため、練習モードも `500ms` に統一（`js/game.js`）

## v1.2.3 — 2026-07-01

### バグ修正・ガイド文言修正
- **タイマー加算漏れの修正**: `Game._startTimer()` が呼ばれるたびに経過時間を加算せず上書きしていたため、「もう一度」ボタンで出題音を聞き直すと直前までの経過時間が消えていた（実質タイムのリセット扱いになっていた）。前回開始時刻からの経過分を `_totalTime` に加算してから計り直すよう修正（`js/game.js`）
- **`guide.html` を実装と整合**:
  - タイマーの説明を「画面表示のタイマー（ゲーム画面に入ってから止まらず動き続ける経過時間）」と「ランキング保存用タイム（出題音再生後から正解までを10問分合計、別計算）」が別物である旨に修正
  - マスク弦の説明を「判定弦（灰色がかっている弦）は対象外」という矛盾した表現から「暗く塗りつぶされている弦（マスクされた弦）が対象外、暖色ハイライトの弦がタップ可能」に修正（前回の指板視認性改善に合わせた更新）
  - 開放弦除外の説明を実際のUI（「開放弦」ボタンの表示が「除外」になっている状態）に合わせて修正

## v1.2.2 — 2026-07-01

### 音再生の調整
- **発音時間を全体的に延長**: タップフィードバック音 `0.5秒→0.7秒`、出題ルート音 `0.6秒→0.8秒`、出題度数音 `0.8秒→1.0秒`。ルート→度数音の再生間隔も `700ms→800ms` に調整
- **後勝ちのモノフォニック再生**: `AudioEngine.playNote()` に直前の音を即座にフェードアウト（20ms）して停止する処理を追加。連続タップ時は後からタップされた音が優先して鳴るように変更（`js/audio.js`）

## v1.1.1 — 2026-06-30

### セキュリティ・バグ修正（コードレビュー対応）

**セキュリティ修正**
- **`server.js` Path Traversal 対策**:
  - `ROOT_WITH_SEP` でセパレータ付き比較（ディレクトリ名プレフィックス誤許可を修正）
  - `decodeURIComponent` でURLエンコードをデコード後に検証
  - NULバイト（`\0`）を含むパスを 400 で拒否
  - `fs.realpath` でシンボリックリンク先を検証

**ゲームロジック修正**
- **`_hasValidAnswer` の走査範囲修正**: 指板全体（0〜MAX_FRET）→ `calcDisplayRange` の表示窓内のみに限定（練習モードで表示窓外の正解を誤検出していたバグを修正）
- **練習モード `judgeStrings` 拡張**: `[0]`（6弦のみ）→ `[0,1,2]`（低音3弦）に変更（5th=G が1弦の表示窓に入らず詰むケースを解消）
- **タイマー管理強化**: `stop()` で `_playTimer` と `_nextQuestionTimer` の両方をキャンセル
- **`_nextQuestion` ハンドル保存**: `setTimeout` の戻り値を `this._nextQuestionTimer` に保存

**その他修正**
- **localStorage guard**: `loadScores()` / `saveScore()` を `try/catch` で包み、iOS Safari プライベートブラウズの `QuotaExceededError` を無音処理
- **touchend/click 二重発火防止**: `suppressNextClick` フラグでタップ直後の click イベントを抑制
- **縦余白タップ誤検出防止**: `_handleTap` で Canvas 上下余白領域のタップを無視

## v1.1.0 — 2026-06-30

### 機能拡張・バグ修正
- **17F対応**: 表示・タップ判定・出題バリデーションを 0F〜17F に統一
  - `MAX_FRET = 17`（旧: 16）
  - `calcDisplayRange` の上限を `MAX_FRET-1 = 16` に修正（最大表示ゾーン = 17F、18F 表示バグを解消）
  - ポジションマークに 17F を追加
- **基準音バリエーション拡張（LV.4 / LV.Max）**: `rootStrings: [0,1,2]`・`rootOctaves: [0,1]` で 6〜4弦 × 0/1 オクターブからランダム選択
- **出題バリデーション**: `_hasValidAnswer()` + 最大 30 回リトライで「18F 以上にしか正解がない」問題を排除
- **開放弦対応**
  - ナット左側に 0F 専用ゾーンを描画、タップ可能
  - フレット番号を `0, 1, 2, 3...` 形式で表示
  - 開放弦を回答対象から除外できる設定をハンバーガーメニューに追加
- **波形デフォルト変更**: `triangle` → `square`
- **2タップ方式**: フェーズ1（Root タップ）→ フェーズ2（度数音タップ）の順を強制
- **縦持ち対応**: CSS rotate 削除、「横向きにしてください」オーバーレイ方式に変更
- **PWA**: Service Worker を `fretboard-v8` に更新
