- パネル高さを一時的に縮小した状態でオーバーフロー発生とスクロール動作を実機ブラウザで確認済み。通常サイズでのメニュー表示・各設定項目の動作に変化がないことも確認
- ハンバーガーメニューのバージョン表示を `v1.5.3` に更新

## v1.5.1 — 2026-07-01

### コードトーン編 Stage 0 のコードレビュー指摘を修正
コードレビュー（`CODE_REVIEW_2026-07-01_chordgame-stage0.md`）で指摘された4点に対応。

- **`_confirmedRoot` のモード間リークを修正**: インターバル編でRootを正解タップした後、度数を答える前にホームボタンで離脱してコードトーン編を開始すると、オレンジのルート確定マーカーが指板上に残留するバグを修正。`ChordGame._nextChord()` の先頭で `fretboard.clearConfirmedRoot()` を呼ぶよう追加
- **`stop()` でコード再生中の音を即カット**: `AudioEngine` に `stopChord()`（`_chordVoices` をフェードアウトして null 化）を追加し、`ChordGame.stop()` から呼ぶことで、ホームボタン離脱時にコード音（最大約1.2秒）が鳴り続ける問題を解消
- **フッタータイマー表示を仕様（タイマーなし）に合わせて非表示化**: `app.js` の `startChordPractice()` で `elTimer` に `hidden` クラスを付与し `startTimerDisplay()` の呼び出しを削除。インターバル編開始時（`startGame()`）は `hidden` を解除して従来通り表示
- **クリア済み構成音の再タップを中立色でフィードバック**: 既に消し込んだ度数を別ポジションで再タップした際、緑（正解）ではなく中立色（`COLOR.neutral` の青系）で表示するよう修正。`Fretboard.showFeedback()` の第3引数に `'neutral'` 状態を追加（`true`/`false` の既存呼び出し元＝インターバル編には影響なし）
- 上記4点はローカルサーバーでの実機ブラウザ確認済み。あわせて既存インターバル編（LV.1の複数問プレイ・タイマー動作・正誤フィードバック）への影響なしも確認
- **Stage 1への申し送り（今回未対応）**: `ChordGame` には出題可解性の検証（`Game._hasValidAnswer` 相当）がない。現状（ルートC固定・低音3弦・maj/minのみ）は手計算上すべて可解だが、ルートやコードタイプを増やすと表示窓内に構成音が収まらない「詰み」が発生し得るため、Stage 1着手時に必ず対応すること
- `sw.js` を `fretboard-v11` に更新（`js/chordGame.js` / `js/audio.js` / `js/app.js` / `js/fretboard.js` の内容変更をキャッシュに反映）

## v1.5.0 — 2026-07-01

### コードトーン編 Stage 0 を新規実装
- **新規 `js/chordGame.js`（`ChordGame` クラス）**: 「未クリア集合」方式の判定ロジックを実装
  - ルート C 固定、メジャー / マイナートライアド（`CHORD_TYPES.maj` / `.min`）のみ、低音3弦（6〜4弦）固定、練習モードのみ（タイマーなし・無限ループ）
  - コード開始時に構成音のピッチクラス集合を保持し、順不同のタップで発見した度数を集合から削除。空になったら 800ms 後に次のコードへ自動遷移
  - 進捗は常時テキスト表示（例:「R ✓　m3 ✓　5th（未）」）。ヒントや表示補助のON/OFFはStage 0の対象外
- **`js/audio.js` のポリフォニック再生バグを修正**: 既存の `playChord(midis, duration)` は内部で `playNote()` を呼んでいたため、単音再生用の後勝ちカット処理（`_activeVoice`）が働いて事実上モノフォニックになっていた（同時に複数音を鳴らし続けられなかった）。単音用ボイス生成処理を `_createVoice()` に共通化した上で、コード再生専用の独立管理（`_chordVoices`）を新設し、複数 `OscillatorNode` を同時に鳴らし続けられるよう修正
- **`index.html` / `js/app.js`**: ホーム画面にコードトーン編セクションを追加し、モード切り替え（インターバル編 ⇄ コードトーン編の表示トグル）を実装。準備中アラートを削除
- **`guide.html` にコードトーン編（Stage 0）の説明セクションを追加**: 遊び方（コードを聴く→構成音を順不同でタップ→次のコードへ）、Stage 0 の対象範囲（練習モードのみ・ルートC固定・メジャー/マイナートライアドのみ・低音3弦）を明記。既存のインターバル編の記述は変更なし
- `sw.js` を `fretboard-v10` に更新、`./js/chordGame.js` をキャッシュ対象に追加
- ローカルサーバーでの実機ブラウザ確認済み（未クリア集合の正解/不正解判定、進捗テキスト更新、コード自動遷移、既存インターバル編〔LV.1の10問完走・スコア保存・ランキング・練習モード・ヒント機能・メニュー設定〕への影響なしを確認）。GitHub Pages への公開は未実施

## v1.4.0 — 2026-07-01

### バグ修正・ヒント表示機能を追加
- **ハンバーガーメニュー表示バグの修正**: メニューを開いたままLVボタンをタップするとメニューが開いたままゲーム画面に遷移し、画面の一部が隠れる不具合を修正。`startGame()` の先頭で `menuPanel` の `open` クラスを外すよう変更（`js/app.js`）
- **詰まった時のヒント表示機能を追加**（LV.Max以外の全LV・練習モード対象）:
  - フェーズ1（Root待ち）: 出題音再生完了直後から5〜7秒（ランダム）無反応でタイムアウトし、表示範囲内・判定対象弦の中でルートのピッチクラスに一致する全ポジションに半透明ドットをフェードイン表示
  - フェーズ2（度数音待ち）: Rootタップ確定直後から5〜7秒無反応でタイムアウトし、同様にターゲット音の全ポジションを表示
  - ヒントの使用有無はスコア・ランキングに一切影響しない（記録しない）
  - `js/fretboard.js`: `showHints()` / `clearHints()` を追加。`requestAnimationFrame` で約400msかけてフェードイン。`_drawCircle()` に半径スケール・不透明度の引数を追加（既存呼び出しは後方互換）
  - `js/game.js`: `_scheduleHint()` でスケジュール時点のフェーズを記憶し、フェーズが変わっていたら発火時に無視（「もう一度」で度数フェーズ中に音を聞き直した際の誤発火防止）。ルート正解時・度数正解時・新しい問題開始時・ゲーム停止時にヒントとタイマーをクリア
  - ハンバーガーメニューに「ヒント」ON/OFFトグルを追加（デフォルトON、`js/app.js` / `index.html`）
- `guide.html` にヒント機能の説明を追記
- **練習モードの次問移行タイマーを統一**: 練習モードのみ正解後の次問移行が `800ms`（通常LVは `500ms`）となっており、`guide.html` の「0.5秒後に自動で次の問題へ進みます」という記述と食い違っていた。意図的な設計差ではなかったため、練習モードも `500ms` に統一（`js/game.js`）

## v1.1.1 — 2026-06-30

### セキュリティ・バグ修正（コードレビュー対応）

**セキュリティ修正**
- **`server.js` Path Traversal 対策**:
  - `ROOT_WITH_SEP` でセパレータ付き比較（ディレクトリ名プレフィックス誤許可を修正）
  - `decodeURIComponent` でURLエンコードをデコード後に検証

- **`_hasValidAnswer` の走査範囲修正**: 指板全体（0〜MAX_FRET）→ `calcDisplayRange` の表示窓内のみに限定（練習モードで表示窓外の正解を誤検出していたバグを修正）
